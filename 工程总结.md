# 即时通讯系统 — 工程总结

## 项目概览

基于 **Qt 5/C++** 的局域网易用即时通讯系统，C/S 架构，包含服务端和客户端两个独立程序。

| 平台 | 编译器 | Qt 版本 |
|------|--------|---------|
| Windows | MinGW 7.3 64-bit | Qt 5.12.10 |

## 通信架构

```
┌──────────┐  TCP:8888   ┌──────────┐
│ Client A │ ←──────────→ │  Server  │
│ (Qt GUI) │   JSON\n     │ (Qt GUI) │
└──────────┘              └──────────┘
                               ↕
┌──────────┐              ┌──────────┐
│ Client B │              │ server.db│
│ (Qt GUI) │              │ SQLite   │
└──────────┘              └──────────┘
```

### 协议：JSON 行协议

每条消息一行 Compact JSON，以 `\n` 分隔。

| 消息类型 | 方向 | 用途 |
|----------|------|------|
| `login` / `login_resp` | C↔S | 登录认证 |
| `register` / `register_resp` | C↔S | 注册 |
| `send_msg` → `recv_msg` + `send_ack` | C→S→C | 消息转发 + 送达回执 |
| `friend_list` | S→C | 推送好友列表 |
| `status_change` | S→C | 好友上下线广播 |
| `add_friend` / `add_friend_resp` / `friend_added` | C↔S | 添加好友（双向） |
| `avatar_update` / `avatar_sync` | C→S→C | 头像变更同步 |

## 功能矩阵

### 服务端

| 功能 | 实现 |
|------|------|
| 多客户端并发 | `QTcpServer` + 每个连接一个 `ClientHandler` |
| 用户管理 | SQLite `users` 表：注册/登录/头像/在线状态 |
| 好友管理 | `friends` 表：双向添加、分组、列表推送 |
| 消息转发 | 在线即时转发 + 离线回执 + 存库 |
| 管理界面 | `ServerMainWindow`：启停、端口、在线客户端树、彩色日志 |

### 客户端

| 功能 | 实现 |
|------|------|
| 登录/注册 | 模式切换 UI：登录 ⇄ 注册表单展开 |
| 密码保护 | 3 次错误锁定 + 验证解锁 |
| 好友列表 | 分组（家人/朋友/同事/其他）+ 在线状态 🟢⚫ + 头像 + 右键移动分组 |
| 文字聊天 | 气泡式消息（自右蓝/他左白）+ QTextTable 表格布局 |
| Enter 发送 | `eventFilter` 拦截键盘：Enter 发送 / Ctrl+Enter 换行 |
| 表情 | `EmojiPicker` 弹窗：36 个 Unicode Emoji |
| 语音识别 | 按住 🎤 录音 → 松开 → 百度 API → 文字填入输入框 |
| 聊天记录 | 本地 SQLite 持久化 + 导出 TXT |
| 头像 | Base64 编码传输 + 好友列表解码显示为 QIcon |
| 消息通知 | 底部通知栏 + 未读自动弹出聊天窗口 |
| 消息回执 | 离线消息提示 ⚠ 对方不在线 |

## 数据库设计

### 服务端 server.db

```sql
CREATE TABLE users (
    account  TEXT PRIMARY KEY,   -- 账号
    nickname TEXT NOT NULL,      -- 昵称
    password TEXT NOT NULL,      -- 密码（明文 ⚠️）
    gender   TEXT DEFAULT '男',
    avatar   TEXT DEFAULT 'default',  -- Base64 数据 URL
    online   INTEGER DEFAULT 0
);

CREATE TABLE friends (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    account       TEXT NOT NULL,       -- 自己的账号
    friendAccount TEXT NOT NULL,       -- 好友的账号
    groupName     TEXT DEFAULT '其他',
    UNIQUE(account, friendAccount)
);

CREATE TABLE messages (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    sender    TEXT NOT NULL,
    receiver  TEXT NOT NULL,
    content   TEXT NOT NULL,
    msgType   TEXT DEFAULT 'text',
    timestamp TEXT NOT NULL
);
```

### 客户端 chat_{账号}.db

```sql
CREATE TABLE messages (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    sender    TEXT NOT NULL,
    receiver  TEXT NOT NULL,
    content   TEXT NOT NULL,
    msgType   TEXT DEFAULT 'text',
    timestamp TEXT NOT NULL
);
```

## 语音识别链路

```
按住按钮 → AudioRecorder.startRecord()
         → QAudioInput → 16kHz PCM → 临时文件

松开按钮 → AudioRecorder.stopRecord()
         → BaiduSpeech.recognize(filePath)
         → POST http://vop.baidu.com/server_api
         → JSON 响应 → result[0]
         → 填入 QTextEdit
```

Token 预先通过 HTTPS OAuth2 获取，有效期 30 天。MinGW 需要 `libssl-1_1-x64.dll` 在 exe 目录下。

## 技术要点

| 要点 | 方案 |
|------|------|
| TCP 异步连接竞态 | `connectToServer()` 后等待 `connected` 信号再发请求 |
| UTF-8 源文件 | MinGW 不需要 BOM（已移除） |
| 单例模式 | `NetworkClient`、`DatabaseManager` 使用饿汉式单例 |
| 好友列表 Widget 提升 | `.ui` 中 QTreeWidget 提升为 `FriendListWidget` 子类 |
| QGridLayout 替代 QFormLayout | 避免 uic 解析时的控件堆叠 bug |
| 全局按钮样式 | `padding: 4px 10px` + 单个按钮 minWidth 覆盖 |
